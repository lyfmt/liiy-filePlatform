# liiy-filePlatform

一个轻量级公开文件中转平台。它使用 Go 标准库实现，只有一个服务进程，不依赖数据库，也不需要额外前端构建流程。

## 功能

- 支持上传文本、图片和任意文件
- 所有访问者都可以直接看到已上传内容
- 文本直接展示，图片直接预览，其他文件显示图标和文件名
- 所有内容都支持下载
- 上传 24 小时后自动删除
- 上传时可选择保留 1 到 24 小时
- key-value 形式展示内容：`key` 为上传名，留空时默认使用文件名
- 上传页与下载页分离，下载页支持按文本、图片、文件筛选
- 支持登录校验，可在首次启动时通过终端生成 `config.yaml`
- 登录 cookie 默认有效期 30 天，支持前端主动退出

## 设计特点

- 单个 Go 二进制，内存占用和部署复杂度都很低
- 文件和元数据直接落盘，不接数据库
- 页面使用服务端模板和原生表单，没有前端框架和打包链路
- 过期清理逻辑由后台定时器和请求前检查共同完成

## 运行

```bash
go run .
```

首次启动如果没有 `config.yaml`，程序会在终端询问：

- 是否开启登录校验
- 是否自定义账号和密码

如果不自定义，程序会自动生成 8 位大小写字母和数字组合的账号密码，并写入 `config.yaml`。之后你可以手动修改该文件中的 `auth_enabled`、`username` 和 `password`，修改后重启服务即可生效。

如果开启登录校验，浏览器登录 cookie 默认有效期为 30 天，前端右上角提供主动退出入口。

默认监听 `8080` 端口，数据存放在项目根目录下的 `data/`。

## 环境变量

- `PORT`: 服务端口，默认 `8080`
- `DATA_DIR`: 数据目录，默认 `./data`
- `MAX_UPLOAD_MB`: 单次上传大小限制，默认 `32`
- `CONFIG_PATH`: 配置文件路径，默认 `./config.yaml`

## 构建

如果需要二进制运行方式，可以先编译：

```bash
go build -o liiy-file-platform .
./liiy-file-platform
```

## 存储结构

运行后会自动创建：

```text
config.yaml     # 运行配置与登录账号密码
data/
  files/        # 实际文件内容
  index.json    # 元数据索引
```

`index.json` 只保存描述信息，实际内容仍然按文件落盘，便于清理和下载。

## config.yaml 示例

```yaml
port: "8080"
data_dir: "./data"
max_upload_mb: 32
auth_enabled: true
username: "Ab12Cd34"
password: "Xy56Za78"
session_secret: "aBcDeF123456GhIjKlMn7890OpQrStUv"
```

说明：

- `max_upload_mb` 修改后，刷新页面即可看到前端提示同步变化，上传接口也会使用新限制
- 单次上传的实际保留时长在前端上传时选择，不写在全局配置中
